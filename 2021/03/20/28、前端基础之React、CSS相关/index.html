JS中函数的call()，aapply()和bind()方法。

相同点：call()、apply()和bind()方法都可以改变函数this对象的指向。
不同点：

  • call()函数的第一个参数是this指向的对象，其余的每一个参数之间需要用都好隔开。
  • apply()函数的第一个参数是this指向的对象，其余的需要传给函数的参数都是放在一个数组里面的。
  • bind()函数最终执行以后返回的是一个函数，必须调用这个函数才会执行。而call()和apply()在改变函数的this指向之后都是立刻执行的。

React的新特性

  • render()函数支持返回数组和字符串。
  • 当React 15在渲染的时候发生错误，就会导致整个React组件崩溃。在React 16版本里面添加了更好的错误处理策略，如果一个错误是在组件渲染或者是声明周期函数中被抛出，整个组件就会从根节点被卸载，而不会影响其他组件的渲染，通过error boundaries进行处理就可以。当组件的render()函数抛出错误的时候就会触发componentDidCatch()函数对错误进行处理。
  • createPortal的出现为组件弹窗，对话框的开发提供了便利，
  • React的Context为组件之间的数据传递提供了一种新的方式，可以避免在每一级组件之间手动的传递数据。

React的特点

  • （1）声明式的视图层，不需要担心数据、状态和视图层交错纵横在一起了。
  • （2）简单的更新流程。开发人员只要定义好UI状态，React会自动将数据渲染在UI中，而且从数据到UI的这个流程是单向的，整个更新过程更加简洁清晰。
  • （3）灵活的渲染实现。React先把视图层渲染成Virtual DOM，是一个普通的JavaScript对象，再结合不同的依赖库渲染成不同终端的UI。
  • （4）React有高效的DOM操作。当组件发生更新的时候React会尽量更新最少的DOM元素，做最少的DOM操作，可以大幅度提升代码执行的性能。

React相关的开发环境以及工具介绍

  • （1）Node.js：Node.js是一个JavaScript运行时，React应用在编译的时候需要的很多依赖，例如Webpack，NPM等都需要使用到Node.js环境。
  • （2）NPM：一个模块管理工具，当React应用在开发过程中需要使用到其他的模块的时候可以通过NPM进行下载和管理。
  • （3）Webpack：一个前端自动化构建工具，可以将程序需要的模块进行打包。
  • （4）Babel：ES6语法的转换器，由于在React开发过程中使用了大量的ES6语法，以及JSX代码。这些代码不能被所有的浏览器兼容，所以需要通过Babel转换成ES5代码供浏览器执行。
  • ESLint：一种JavaScript代码风格检查工具，有助于开发人员规范代码编写。

JSX简介

JSX是一种用于描述UI的JavaScript的扩展语法，React使用这种代码来描述UI组件。组件时React的核心，是React应用的基石。

使用Class定义React组件需要注意的事：

  • 组件类class必须要继承React.Component。
  • 组件类内部必须定义render方法，该方法返回的是代表该组件UI的React元素。

React的组件分类

  • 有状态组件：一个组件内部的状态会发生变化，需要使用state来保存组件状态的时候，这个组件就是有状态组件。
  • 无状态组件：一个组件只接收props作为参数而没有内部的state，这个组件就属于无状态组件。

如果需要一些数据并且在组件内部有一些对数据的操作逻辑，就需要使用有状态组件。如果只是接收数据进行渲染，则可以使用无状态组件。

React组件的样式

  • 外部样式：React组件中要使用className来代替class。
  • 内联样式：内联样式是CSS in JS的一种写法，样式的属性名称需要按照驼峰的方式命名。

React组件的生命周期

组件生命周期的三个阶段：挂载阶段、更新阶段和卸载阶段。

挂载阶段
  • constructor：组件的构造方法，当组件被创建的时候就会调用这个方法，该构造方法接收props作为参数，在constructor()方法中还可以进行state的初始化操作以及一些事件的绑定操作。
  • componentWillMount：该方法是在组件被挂载到DOM之前进行操作的，在这个方法中调用this.setState()不会引起组件的重新渲染。
  • render：该方法是组件唯一必须实现的方法，render()函数是一个纯函数，最终返回的是一个UI组件。
  • componentDidMount：该方法是在组件被挂载到DOM之后进行调用的，而且只会被调用一次。可以在该方法中向服务器请求数据，一些依赖DOM节点操作的逻辑可以放在这个函数中执行。
更新阶段
  • componentWillReceviceProps(nextProps)：该方法只有在接收props，组件发生更新的时候才会调用。
  • shouldComponentUpdate(nextProps, nextState)：该方法会返回一个布尔值，true表示组件需要被更新，false表示组件不需要被更新。在这个方法中一般需要比较nextProps、nextState和组件当前的props、state来决定组件是否有必要进行更新。
  • componentWillUpdate(nextProps, nextState)：在render函数执行之前执行。
  • componentDidUpdate(prevProps, prevState)：在组件更新之后被调用。
卸载阶段
  • componentWillUnmount()在组件卸载之前会被调用这个函数。在这个方法中可以执行一些清除的操作，例如解除之前绑定的事件函数等。

只有类组件才有生命周期函数，函数组件时没有生命周期函数的，所以不能在函数组件中使用生命周期函数。

React中的事件处理

React事件中需要注意的地方：

  • 事件的命名方式需要采用驼峰的命名方式，而不是DOM元素的小写字母的方式。
  • 处理事件的响应函数需要以对象的方式赋值给事件属性。

React中的事件是合成事件，并不是原生的DOM事件。可以通过在事件处理函数中显式的调用preventDefault()来阻止事件的默认行为。

受控组件和非受控组件

  • 受控组件：当一个组件的值是由React进行管理的，那么这个组件就是受控组件。例如input、textarea元素的受控原理是，通过表单元素的onChange事件监听值的变化，将这种变化同步到React中的state中，通过表单元素的value属性来设置元素的值，列表元素是通过在select上定义value决定哪一个option处于选中的状态，复选框和单选框控制其checked状态实现组件控制的。
  • 非受控组件：非受控组件指的是表单元素的状态是由表单自己进行管理而不是通过React进行管理，这样的组件称为非受控组件。React中提供了一个特殊的属性ref，用于引用React组件或者DOM元素实例。

React 16的新特性

  • React 16之前render函数只能返回一个元素，React 16之后render函数可以返回一个数组或者一个字符串。
  • React 16之前当组件中运行出错之后就会阻塞整个应用的渲染，React 16之后引入了错误处理机制，当组件中发生错误的时候这个组件会从组件树中卸载，可以避免整个应用程序崩溃。
  • React 16中新添加了Portals，任何组件都可以将弹窗组件渲染在根本节点上，以便于弹窗的显示。

修改state

调用setState()的时候，组件的state并不会直接改变，setState()只是会把要修改的状态放在一个队列中，React会将多次setState()的修改状态合并成一次状态修改。setState()方法是一个异步的方法，props的更新也是一个异步的过程。

为什么直接修改this.state是无效的？

当调用setState()修改state的时候是通过一个队列机制实现state更新的，执行setState()的时候会将需要更新的state的值合并以后放在一个状态队列里面，而不会立刻更新state，状态队列可以批量更新state，如果不通过setState()而是使用this.state直接进行更新的话，就不会放在队列里面，就不能实现state的更新。

关于setState()的总结：

  • 开发过程中需要调用setState()去更新state，而不能直接去操作this.state，React是把state当作不可变对象。state中的某一个状态发生更新的时候应该去重新创建这个对象，而不是更新这个对象的某一个属性值。
  • 调用setState()修改state的时候不是马上生效的，setState()方法是异步的，执行完setState()之后state的值也并不是最新的值。
  • 多个顺序执行setState()不是同步的一个一个的执行，会一个一个加入队列，最后进行批处理，实现state的更新。

setState()与不可变对象

根据状态类型可以分为以下三种情况：

  • 状态的类型是不可变类型（数字，字符串、布尔值、null、undefined）：这一类的值在修改的时候直接给需要修改的状态赋一个新值就可以。
  • 状态的类型是数组：使用preState()、concat()结合之前的数组和要修改的值创建一个新数组，或者使用ES6数组解构赋值创建一个新数组。（不要使用push、pop、shift、unshift、splice因为这几个方法是在原有数组的基础上进行修改的，而不是创建了一个新的数组）
  • 状态类型是普通对象：使用ES 6的Object.assign()或者使用对象扩展语法创建一个新的状态对象。

如何知道state的状态已经被更新

  • 在调用setState()函数的时候，传入一个回调函数，当setState()函数已经将state更新之后可以在回调函数中获取到最新的state。
  • 在setState()的第一个参数传入一个函数，该函数会在执行的时候被压入调用栈中，在state真正改变之后会按顺序回调栈里面的函数，函数的第一个参数就是上一次更新之后的state。这样就可以确保在下一次操作中获取的值就是预期的state。

组件与服务器通信

  • 组件挂载阶段的通信：在组件挂载阶段使用componentDidMount()进行与服务器的通信时最好的，因为在componentDidMount()执行的时候组件已经处于挂载状态了，这个时候直接操作DOM是安全的。而componentWillMount()这个函数会被调用两次，一次在服务器上，另一次是在浏览器。而componentDidMount()方法只会被调用一次，使用componentDidMOunt()可以避免发送多余的请求。
  • 组件更新阶段的通信：在这个阶段可以在componentWillReceviceProps()函数中与服务器进行通信。

组件之间的通信

  • 父子组件之间的通信：父组件通过子组件的props与子组件进行通信，实现数据传递。子组件向父组件发送数据也可以通过父组件的props来完成，父组件可以通过子组件的props传递给子组件一个回调函数，子组件要改变父组件的数据的时候调用这个回调函数就可以实现子父组件通信。
  • 兄弟组件通信：兄弟组件之间不能直接进行数据传递，需要通过状态提升实现兄弟组件之间的数据传递，可以将需要共享的数据状态提升到离他们最近的共同的父组件里面，任何一个组件可以通过父组件传递的回调函数来修改共享的数据状态。
  • Context：当组件层级很多需要跨层级传递数据的时候，使用props传值的方式会比较繁琐，React的Context上下文可以让任意层级的组件都可以获取父组件中的状态和方法，Context的使用方法是：在提供Context的组件内部新增一个getChildContext的方法，返回一个Context对象，然后再组件的childContextTypes属性上定义context对象的属性类型信息，在子组件内部可以通过this.context的方式获取从父级组件中传递的数据和方法。程序实例：

特殊的ref

ref的使用场景

  • 在DOM元素上使用ref：ref接收一个回调函数作为值，在组件被挂载或者被卸载的时候，回调函数都会被调用，在组件挂载的时候回调函数会接收当前的DOM元素作为参数，在组件被卸载的时候，回调函数会接收null作为参数。
  • 在组件上使用ref：在组件上使用ref的时候，ref的回调函数接收的参数是当前组件的实例。可以提供一种外部操作组件的方式。只能为类组件定义ref，不能为函数组件定义ref。
  • 父组件访问子组件的DOM节点：在子组件的DOM元素上定义ref，ref的值是父组件传递给子组件的回调函数回调函数可以通过一个自定义属性传递，这样父组件就可以获取到这个DOM元素。

虚拟DOM和性能优化

前端每一次对DOM进行的操作都会引起浏览器的重绘和重排，所以虚拟DOM的出现就是为了减少DOM操作，提升代码运行的性能。虚拟DOM是使用JavaScript描述的一种DOM结构。当我们需要操作DOM的时候可以先操作虚拟DOM，再将虚拟DOM渲染。因为操作JavaScript对象会比直接操作DOM元素速度更快。

Diff算法

Diff算法就是将前后两个虚拟DOM进行对比，找出需要更新的DOM元素，从而实现更新。Diff算法比较的是新的虚拟DOM和旧的虚拟DOM。最后将Diff算法的结果更新到真实DOM上。

Diff算法的两个基本假设：

  • 如果两个元素的类型不同，那么他们将会生成两个不同的树。
  • 为列表中的列表元素设置key属性，判断用key标识的元素在render过程中是否发生了变化。

Diff算法的工作流程：

  • 当根节点不同时：当两个根节点不同的时候，React会新的树和旧的树完全不同，不会再比较该节点以下的其他节点和属性，而是把整个树全都销毁进行重建，在这个过程中旧的DOM元素会被销毁，新的DOM元素会被插入到树里。旧的React组件会被卸载，会调用组件的componentWillUnmount()方法，新的组件实例的componentWillMount()和componentDidMOunt()方法会被调用。在重新创建DOM树之后，又会整体更新到真实的DOM树中。
  • 当节点是相同的DOM元素时：React会保留根节点，然后比较根节点的属性，最后只更细那些发生变化的属性。
  • 当根节点是两个相同的组件类型：如果两个根节点是相同的两个组件，对应的组件实例不会被销毁，只是会执行更新的操作，将变化同步到虚拟DOM上，在这个过程中组件实例的componentWillRevecieProps()和componentWillUpdate()会被调用。

关于React开发过程中的性能优化

  • 使用生产环境的版本库，生产环境的库稳定，体积小，执行速度快。
  • 避免不必要的条件渲染，shouldComponentUpdate()方法可以判断一个组件是否有必要进行更新，该函数返回布尔值，true表示需要更新，false表示不需要更新。
  • 使用key，Diff算法会根据key值来判断拥有相同key值的元素是否能是同一个元素，使用稳定不变，唯一确定的key值可以加快Diff算法的对比速度，从而提升效率。

高阶组件

高阶函数以函数作为参数，最后会返回一个新的函数。高阶组件以一个React组件作为参数，最后会返回一个新的组件。

高阶组件的使用场景：

  • 操纵props：高阶组件可以在被包装的组件接收到props之前，先拦截props，可以对props进行增加，删除或者修改的操作，然后将处理之后的props传递给被包装的组件。
  • 通过ref访问组件实例：高阶组件通过ref获取被包装组件的实例的引用，就可以操作被包装组件的属性和方法。
  • 组件状态提升。

React路由

React应用是一种单页面应用，React Router可以让wen应用根据不同的路由渲染不同的组件，从而解决复杂的应用场景。

React Router路由的安装：

React Router包含了三个库：react-router，react-router-dom和react-router-native。

React Router通过Router和Route两个组件完成路由的功能。只需要一个Router实例，所有的路由配置组件Route都会定义为Router的子组件。BrowserRouter使用HTML5的history API实现UI与URL的同步。HashRouter使用URL的hash实现UI与URL的同步。

路由配置

  • path：每一个Route都需要定义一个path属性，当使用BrowserRouter时，path用于描述Route匹配的URL的pathname。当使用HashRouter时用于匹配URL的hash。
  • match：当路由匹配的时候React会创建一个macth对象作为props中的一个属性传给被渲染的组件。该对象包含了以下4个属性：
    (1)params：路由的path可以传递参数，params就是从URL中解析出来的参数。
    (2)isExact：用于设置是否完全匹配。
    (3)path用于构建嵌套路由的时候会用到。
    (4)url
  • Route渲染组件的方式
    （1）component：component的值是一个组件，当路由匹配的时候就会渲染对应的组件。
    （2）render：render的值是一个函数，最后会返回一个React元素。
    （3）children：是一个函数，返回一个React元素，无论路由是否匹配最后返回的元素都会被渲染。

Redux可预测的状态管理机

Redux的三大原则：

  • 唯一数据源：Redux应该只维护一个全局的状态对象，唯一数据源是一种集中管理应用状态的方式。
  • 保持应用状态只读：不能直接修改应用状态，而是需要发送一个action，action描述了如何修改应用程序的状态。
  • 应用状态的改变只能通过纯函数完成，reducer是一个纯函数，reducer在接到action之后，不能直接修改原来的对象，而需要创建一个新的状态对象返回。

纯函数的概念

  • 对于同样的参数值，函数返回的结果总是相同的。
  • 函数的执行不会产生其他的副作用。

Redux的组成

  • action：action是Redux中的信息载体，是store中唯一的数据来源。把action发送给store必须通过store的dispatch方法。
  • reducer：reducer会根据action做出响应，决定如何修改状态state。
  • store：是action和reducer之间的桥梁，用于保存应用状态。可以通过getState()方法获取应用状态，通过dispatch(action)方法发送更新状态的action。可以通过subscribe(listener)注册监听函数，监听应用程序状态的变化。

在React中使用Redxu

  • connect()：用于把React组件和store连接起来，生成一个容器组件，负责数据管理和业务逻辑。
  • mapStateToProps：用于将store中的状态映射到组件里面，作为组件的props。
  • mapDispatchToProps：用于将需要的action映射到组件的props中。
  • Provider组件：Provider组件作为容器组件，接收一个store属性，把store的属性保存在context中，传递给子组件。在使用的时候将Provider组件作为跟组件，这样内层的子组件才能通过context获取到store对象。

使用Redux DevTools可以在React开发过程中调试Redux.

React 16的生命周期

初始化阶段：

  • constructor()
  • getDefaultProps(): 获取Props的默认值
  • getInitialState()：获取初始的state

挂载阶段：

  • static getDerviedStateFromProps(props, state)
  • render()
  • componentDidMount()

getDerivedStateFromProps：组件每次被 rerender的时候，包括在组件构建之后(虚拟 dom之后，实际 dom挂载之前)，每次获取新的 props或 state之后；每次接收新的props之后都会返回一个对象作为新的 state，返回null则说明不需要更新 state；配合 componentDidUpdate，可以覆盖 componentWillReceiveProps的所有用法

更新阶段：

  • staticgetDerivedStateFromProps(props, state)
  • shouldComponentUpdate()
  • render()
  • getSnapshotBeforeUpdate(prevProps, prevState)
  • componentDidUpdate()

卸载阶段：

  • componentWillUnmount()

错误处理：

  • componentDidCatch()

在React 16的生命周期中弃用了之前的componentWillMount()、componentWillReceviceProps()和componentWillUpdate()这三个生命周期函数。新增了getDerviedStateFromProps()、getSnapshotBeforeUpdate()来代替之前启用的这三个函数。

setState()是同步的还是异步的？

在React中对于连续多次调用setState()更新的操作会进行合并，如果想立即使用上次setState()更新的结果作为下一次setState()的参数，可以让setState()接收一个函数而不是一个对象，这个函数将上一次调用setState()更新之后的state作为参数。

React是如何实现自己的事件机制的？

React的事件并不是绑定在真实的DOM节点上，而是通过事件代理机制在最外层的document上对事件进行统一分发。

React事件机制的流程：

组件挂载和更新阶段：

  • （1）判断是否有新增或者删除的事件，然后分别调用事件注册和卸载方法。
  • （2）调用EventPluginHub进行事件存储。
  • （3）获取document对象。
  • （4）给document注册原生事件回调为dispatchEvent(统一的事件分发机制)

事件初始化阶段：

  • （1）EventPluginHub负责管理React合成事件的回调函数，将回调函数存储在ListenerBank中，另外还存储了负责合成事件的plugin。
  • （2）获取绑定事件的元素的唯一的key值。
  • （3）将回调函数根据事件类型以及事件绑定元素的key存储在listenerBank中。

触发事件过程：

  • （1）触发document原生事件的dispatchEvent
  • （2）获取到触发这个事件的最深一级的元素
  • （3）遍历这个元素所有的父元素，依次对每一级元素进行处理。
  • （4）构造合成事件，存放在事件队列中，遍历事件队列。
  • （5）执行合成事件，最后释放完成的事件。

为何React需要自己绑定事件？

因为React中的事件调用是直接调用的，并没有指定调用的组件，所以需要手动绑定事件的this。

React事件和原生事件的区别？

  • React事件使用驼峰命名，而不是全小写
  • 在React中不能返回一个fasle来阻止事件的默认行为，必须显式的调用preventDefault()

React和原生事件的执行顺序是什么？可以混合使用吗？

React中所有的事件都式由document统一进行分发，当真实的DOM触发事件之后，通过事件冒泡过程到达document，才会对React事件进行处理。这个过程比较长，所以是原生事件先执行，然后执行React事件。不可以混合使用。

虚拟DOM一定比普通的DOM快吗？

如果是首次渲染的话，虚拟DOM并不会比普通DOM快。虚拟DOM的优势在于React的Diff算法和批处理策略。也就是说虚拟DOM应用在判断DOM节点更新的时候比普通的DOM快。

虚拟DOM中的$$typeof的作用是什么？

$$typeof是一个Symbol类型的变量，作用是防止XXS攻击。

对于React Hooks的理解？

React Hooks有自己提供的四个钩子函数，开发人员也可以自己编写钩子函数。React约定钩子函数命名以use开头，便于识别。React提供的四个钩子函数：

  • useState()：作用是为函数组件引入状态，纯函数本身不能有状态，所以需要把状态放在钩子函数里面。
  • useContext()：共享状态，当两个组件之间需要共享状态的时候可以通过useContext实现。在钩子函数中引入Context对象，实现状态的共享。
  • useReducer()：action钩子用于引入Reducer功能。
  • useEffect()：用于进行其他的操作，例如从服务器获取数据。

flex布局的理解

flex-direction决定主轴的方向：

  • row：（默认值）水平方向，起点在左端。
  • row-reverse：水平方向，起点在右端。
  • column：垂直方向，起点在顶端。
  • column-reverse：垂直方向，起点在底端。

flex-wrap决定当一行放不下内容的时候如何进行换行：

  • nowrap：（默认）不换行。
  • wrap：换行，第一行在上面。
  • wrap-reverse：换行，第一行在下面。

justify-content：用于定义在主轴上的对齐方式

  • flex-start：（默认）左对齐
  • flex-end：右对齐
  • center：居中
  • space-between：两端对齐，每一块之间的距离都相等。

AMD和CMD的区别

  • AMD推崇依赖前置，CMD推崇就近依赖。
  • AMD是加载完成之后就立即执行，CMD加载完成之后是延迟执行。

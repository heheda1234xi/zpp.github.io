WebScoket详解

一、问题背景

在WebSocket出现之前，当需要与服务器之间保持通讯以保持双方的信息同步。通常这样的持久通讯在不刷新页面的情况下进行，对用户并不可见。在WebSocket出现之前的几种解决办法如下：

1、传统轮询

2、长轮询

传统的轮询方式的缺陷在于每一次请求的时候程序都会创建一个新的HTTP请求，但是并不是每一次请求都会返回新的数据，如果发送的请求数量太多的话，服务器的压力就会很大。所以就出现了一种新的解决办法：长轮询。

长轮询的基本思想是：当客户端每一次发送请求之后，服务器上会检查是否有数据更新，如果有更新的话就会返回更新的数据并且结束本次的请求。如果服务器没有数据更新的话就hold此次的请求，直到有数据更新的时候再返回相应的数据。这种长轮询方式需要通过设置一个较大的HTTP Timeout来实现。程序示例：

长轮询可以有效的避免像传统轮询那样浪费网络带宽但是长轮询是以消耗服务器资源为代价的，尤其是像Apache + PHP这样的服务器由于有worker threads数目的限制，当长轮询数量很多的时候服务器就不能对新请求进行相应。

3、服务器发送事件（Server-Sent Event）

服务器发送事件的方式可以实现从服务器单向到客户端的数据通信。客户端可以自动的获取数据更新，不需要向服务器发送请求。服务器会通过事件流产生并推送事件。事件流对应的MIME类型为text/event-stream，包含了四个字段：event，data，id和retry。event表示事件类型，data表示消息的内容，id用于设置客户端 EventSource 对象的 last event ID string 内部属性，retry指定了重新连接的时间。

4、WebSocket

WebSocket是将TCP协议的Socket(套接字)应用在了web page上。让通信双方建立一个保持连接的通道。WebSocket协议是借用HTTP协议的101 switch protocol来实现协议转换的，从HTTP协议切换成WebSocket协议。

在WebSocket协议中，服务器可以向客户端发送消息，客户端也可以向服务器段发送消息，属于服务器推送技术的一种。

WebSocket的特点：

  • 1、建立在TCP协议上，在服务器端实现比较容易。
  • 2、与HTTP协议有很好的兼容性。
  • 3、可以发送文本信息，也可以发送二进制信息。
  • 4、没有同源限制，客户端可以与任意服务器进行通信。
